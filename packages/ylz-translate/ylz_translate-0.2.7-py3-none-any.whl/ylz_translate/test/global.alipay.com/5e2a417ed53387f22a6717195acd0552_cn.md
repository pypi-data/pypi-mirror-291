结算详情报告
=========================

2024年5月21日 08:14

结算详情报告包含了结算周期内所有交易的结算详细信息。在收到结算银行回单的次日，该报告会与结算汇总报告一起生成。您可以通过 [SFTP服务器](https://global.alipay.com/docs/ac/reconcile/perform#LYsyS) 或 [支付宝全球商家平台](https://global.alipay.com/docs/ac/reconcile/perform#jfNCL) 获取此报告。

报告详情
==============

结算详情报告由三部分组成：头部信息、交易详情以及文件末尾的<END>标记。头部信息由参数名称表示，而结算详情则由这些参数的对应值表示。
以下是结算详情报告中的参数及其描述的表格：

| **参数** | **描述** |
| --- | --- |
| settlementBatchId | **必填** 字符串（64位）：收单方为识别一个结算周期分配的唯一ID，对应于结算汇总参数中的结算批次号。如果收单方未分配此ID，支付宝将生成此ID。 |
| customerId | **必填** 字符串（64位）：收单方为识别商家分配的唯一ID。 |
| acquirer | **必填** 字符串（64位）：处理交易的收单机构名称。这里的收单机构特指为商家提供收单和资金结算服务的机构。 |
| acquirerReferenceNo | **可选** 字符串（64位）：非支付宝收单方为交易分配的唯一ID。当交易由非支付宝收单方结算时，此参数指定。 |
| referenceMerchantId | **可选** 字符串（64位）：主商家为其子商家分配的，直接向用户提供服务或商品的子商家的唯一ID。此字段的值需要与原始支付或授权请求中的*referenceMerchantId*字段相同。 |
| referenceStoreId | **可选** 字符串（64位）：属于商家的店铺的唯一ID，由主商家为其子商家分配。 |
| 交易ID | **必填** 字符串（64位）支付宝为识别一次交易分配的唯一ID。此参数值遵循以下规则：* 当 *transactionType* 为 `PAYMENT`, `CANCEL` 或 `AUTHORIZATION` 时，此参数值与 *paymentId* 相同。* 当 *transactionType* 为 `CAPTURE` 时，此参数值与 *captureId* 相同。* 当 *transactionType* 为 `REFUND` 时，此参数值与 *refundId* 相同。* 当 *transactionType* 为 `DISPUTE` 时，此参数值与 *disputeId* 相同。 |
| 原始交易ID | **可选** 字符串（64位）原始交易的标识符。* 当 *transactionType* 为 `PAYMENT`, `CANCEL` 或 `AUTHORIZATION` 时，此参数值为空。* 当 *transactionType* 为 `VOID` 时，此参数值与原始授权的 *paymentId* 相同。* 当 *transactionType* 为 `REFUND` 时，此参数值与原始支付或授权的 *paymentId* 相同。* 当 *transactionType* 为 `CAPTURE` 时，此参数值与原始授权的 *paymentId* 相同。* 当 *transactionType* 为 `DISPUTE` 时，此参数值与原始授权的 *paymentId* 相同。 |
| 交易请求ID | **必填** 字符串（64位）商家为识别交易请求分配的唯一ID。对于支付，此参数值与 *paymentRequestId* 相同；对于退款，此参数值与 *refundRequestId* 相同。当 *transactionType* 为 `DISPUTE` 时，此参数值为空。 |
| referenceTransactionId | **可选** 字符串（64）由直接向用户提供商品或服务的商家分配的唯一交易ID，用于识别交易。当*transactionType*为`CANCEL`，`CAPTURE`或`DISPUTE`时，此参数的值为空。 |
| paymentMethodType | **必填** 字符串（64）处理支付的支付方式。请参考[支付方式](https://global.alipay.com/docs/ac/ref/payment_method)查看有效值。此参数的值与原始支付或授权请求中的*paymentMethodType*相同。 |
| pspName | **可选** 字符串（64）Alipay+ 移动支付提供商的名称，例如GCash。此参数的值与原始支付或授权请求中的*pspName*相同。 |
| transactionType | **必填** 字符串交易类型。可能的值包括：* `PAYMENT`：表示支付过程。 * `REFUND`：表示退款过程。 * `CANCEL`：表示取消过程。 * `AUTHORIZATION`：表示授权过程。 * `CAPTURE`：表示扣款过程。 * `VOID`：表示取消授权。 * `DISPUTE`：表示争议过程。 * `REFUND_REVERSAL`：表示因支付方式无法正确处理退款而反向退款，资金返回到您的账户。 * `SETTLEMENT_FEE`：表示由收单方收取的结算费用。费用详情由*settlementAmountValue*和*settlementCurrency*参数表示。 * `DISPUTE_REVERSAL`：表示成功辩护的争议被反向处理。 * `COLLATERAL_WITHHOLDING`：表示抵押资金的扣留。 * `RESERVE_WITHHOLDING`：表示储备资金的扣留。 * `RESERVE_RELEASE`：表示储备资金的释放。 * `COLLATERAL_RELEASE`：表示抵押资金的释放。 |
| 支付时间 | **必填** 字符串 (64) | 对应交易成功完成的日期和时间，格式为 ISO 8601 标准定义的 `YYYY-MM-DDTHH:MM:SS+hh:mm`。 |
| 结算时间 | **必填** 字符串 (64) | 结算文件生成的日期和时间，格式为 ISO 8601 标准定义的 `YYYY-MM-DDTHH:MM:SS+hh:mm`。 |
| 产品代码 | **必填** 字符串 | 支付宝与商家之间签订的支付产品类型。有效值为：* `AGREEMENT_PAYMENT`：表示自动扣款支付。 * `CASHIER_PAYMENT`：表示收银台支付。 该参数值与原始支付或授权请求中的 `productCode` 值相同。 |
| 交易金额值 | **必填** 字符串 (16) | 交易金额的主要单位（例如，对于 USD 为美元，对于 JPY 为日元）。当使用 Interchange++ 定价模型时，发卡行的交换费和卡组织的方案费可能在后续的结算批次中收取，此时此字段的值为 `0`。 |
| 交易货币 | **必填** 字符串 (3) | 交易金额值的货币，使用 ISO 4217 标准定义的 3 位货币代码。 |

错误更正信息
------------------------
在以下情况下，您获取的结算报告可能包含错误更正信息：
### 收单机构：恒生电子  
收单机构恒生电子仅提供最终的结算总额。因此，支付宝为您计算每笔交易的结算详情。由于计算方法的差异，支付宝与恒生电子计算出的总结算数据可能不同，故在结算报告中，支付宝提供了金额计算差异的校正信息。校正信息显示在报告的倒数第二行，由_settlementAmountValue_、_feeAmountValue_和_taxFeeAmountValue_表示。例如：

*   如果恒生电子和支付宝计算的总_settlementAmountValue_分别为`37164`和`37140`，则_settlementAmountValue_的校正信息为`24`。
*   如果恒生电子和支付宝计算的总_feeAmountValue_分别为`760`和`760`，则_feeAmountValue_的校正信息为`0`。
*   如果恒生电子和支付宝计算的总_taxFeeAmountValue_分别为`76`和`100`，则_taxFeeAmountValue_的校正信息为`-24`。

最终向您的结算金额以恒生电子提供的结算金额为准。如需了解收单机构为恒生电子时的结算报告详情，请参阅[示例2](#c6DD1)。
### 交互式++计费模型
交互式++是一种计费模型，它包括处理每笔交易时相关发卡银行、卡组织和收单机构收取的费用。这种计费模型透明，因为它提供了最详细的费用分解。交互式++计费模型包括以下三种费用类型：

1. **交换费**：由发卡银行收取的费用。
2. **卡组织费**（first+）：由卡组织收取的费用。
3. **收单费**（second+）：由收单机构收取的费用。

当收单机构是2C2P，并且使用交互式++计费模型时，结算报告可能包含错误修正信息。这些错误修正信息表示应收取的费用与实际收取的费用之间的差异，体现在结算报告的倒数第二行的`feeAmountValue`字段中。

- 如果`feeAmountValue`的值为正，意味着应收取的费用少于实际收取的费用，收单机构需要向您退还差额费用。
- 如果`feeAmountValue`的值为负，意味着应收取的费用多于实际收取的费用，您需要向收单机构支付差额费用。

收单机构不定期将错误修正信息发送给支付宝。支付宝通过结算报告将这些错误修正信息传递给您。
此外，在Interchange++计费模型下，收取的Interchange++费用可能存在延迟。例如，对于2023年1月3日发生的交易，发卡银行收取的交换费没有与其它交易在同一批次结算，而是在下一个批次结算中收取。有关包含Interchange++计费模型下错误修正信息的结算报告的更多信息，请参阅[示例3](#TqQKv)。

报告示例
==========

示例1（标准报告）
--------------------------
在2018年12月25日的交易日，商户ID为102218800000000\*\*\*\*的商家有两笔交易，一笔是付款，另一笔是退款。在支付宝与您结算并收到银行收据后，支付宝生成了一个CSV格式的结算详情报告。报告详情如下：

